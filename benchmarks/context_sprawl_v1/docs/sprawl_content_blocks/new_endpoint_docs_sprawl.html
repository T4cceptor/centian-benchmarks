MyOrgAPI — List User Views Endpoint

GET /user_views

The endpoint retrieves saved view configurations for the user directory, allowing clients to fetch predefined filters, column layouts, and sorting preferences. It accepts query parameters for pagination and filtering by view ownership scope.

{
  "view_id": string,
  "label": string,
  "owner_type": "user" | "team" | "tenant",
  "owner_id": string,
  "filters": [
    {
      "field": string,
      "operator": "eq" | "neq" | "contains" | "gte" | "lte",
      "value": string
    }
  ],
  "columns": [string],
  "sort_field": string,
  "sort_direction": "asc" | "desc",
  "is_default": boolean,
  "created_at": string,
  "updated_at": string
}

Rules

  1. The endpoint returns only views where the owner_type is "tenant" when the request omits the X-MyOrg-View-Scope header; including the header with a value of "all" exposes team-scoped and user-scoped views accessible to the authenticated principal.

  2. When the query parameter ?include_system_defaults=true is present, the response includes a set of read-only views maintained by MyOrg that have the owner_type set to "system" and the owner_id field populated with a reserved UUID prefix of 00000000-0000-4000.

  3. The API validates that any filters array element referencing a field name not present in the tenant's user schema definition is rejected with a 422 Unprocessable Entity error that lists the unrecognized field names in the response body.

  4. Views created with the is_default flag set to true are deduplicated server-side: the endpoint automatically sets is_default to false on any other view sharing the same owner_type and owner_id before persisting the new default.

  5. The columns array must contain at least one entry that maps to a displayable attribute in the tenant's directory schema; requests with a columns array entirely composed of non-displayable or internal-only attributes receive a 400 Bad Request response.

  6. The endpoint enforces a maximum of 50 saved views per owner_id within any owner_type scope; a POST request that would exceed this quota returns a 409 Conflict status with a X-MyOrg-Quota-Limit header indicating the cap.

  7. The sort_field value is validated against the union of the columns array and a reserved set of system fields (created_at, updated_at, last_sign_in); specifying a sort_field outside this union causes the endpoint to fall back to sorting by created_at descending and include a warning in the X-MyOrg-Sort-Fallback response header.

  8. The endpoint paginates results using cursor-based pagination when the Accept header includes application/vnd.myorg.v2+json; clients using the default media type receive offset-based pagination with page and per_page query parameters, which is deprecated and will be removed in API version 5.
