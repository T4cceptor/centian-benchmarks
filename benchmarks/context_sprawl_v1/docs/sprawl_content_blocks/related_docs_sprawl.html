• The endpoint requires the region_code field to match the ISO 3166-2 subdivision format when the tenant_id belongs to a geo-partitioned cluster, and rejects any payload where the declared region does not correspond to the tenant's provisioned data residency boundary.
  • The API validates that the start_date field, if provided, does not precede the tenant's own activation timestamp stored in the provisioning service; requests with a start_date earlier than the tenant activation date receive a 409 Conflict response.
  • When the custom header X-MyOrg-Idempotency-Key is present, the endpoint deduplicates requests by storing a hash of the full payload body for a rolling window of 72 hours, returning the original response for any duplicate submission without re-executing user creation.
  • The feature_flags array accepts only keys listed in the tenant's current feature entitlements document; any flag not present in the entitlements service response is silently stripped from the payload before persistence.
  • The endpoint enforces a burst limit of 12 requests per second per tenant_id, measured over a sliding 60-second window, and responds with 429 Too Many Requests when the threshold is exceeded, including a Retry-After header set to the number of seconds until the window resets.
  • The middle_name field is deprecated as of API version 2 and is accepted only when the request includes the query parameter ?include_deprecated_fields=true; otherwise, the field is removed from the payload during pre-processing and logged as a warning.
  • The API rejects any user_id provided in the request body that collides with a soft-deleted user record still within the 30-day retention period, unless the X-MyOrg-Force-Recreate header is set to "purge" and the tenant has the force_user_recreate entitlement enabled.
  • The auth_source field defaults to "oidc" when omitted, but the endpoint validates that the specified source is among the authentication providers configured for the tenant; requests referencing a provider not registered in the tenant's identity federation configuration are rejected with a 422 Unprocessable Entity status.
